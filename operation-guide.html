一、先看这一页

智慧村务系统是安装在电脑上的村务管理软件。 日常使用时，可以把它理解成：一台电脑负责保存和管理业务数据；如果办公室需要多台电脑一起使用，可在授权后切换为局域网版。

适合谁看

  • 村委、社区、街道的一线录入人员
  • 负责安装、初始化、备份的管理员
  • 需要配置单机版 / 局域网版的实施人员

教程覆盖内容

  • 首次安装和默认账号登录
  • 系统信息、授权、备份和离线升级
  • 居民、务工、风险排查、特殊人群、机构、调解、耕地、报表、待办

不建议跳过

  • 首次上线前先配置字典管理
  • 批量导入前先做数据备份
  • 切换局域网版前先在系统信息里确认授权状态
管理员必知： 当前正式安装版默认是本机单机使用；若要让办公室多台电脑共同访问，需先在“系统信息”里导入与当前电脑匹配的局域网授权，再切换到局域网版并按提示重启。

二、安装与登录

1. 首次安装

  1. 运行安装包，按提示完成安装。
  2. 建议把程序安装在固定位置，后续升级时尽量保持同一安装目录。
  3. 首次启动后，系统会自动准备基础数据，等待完成后进入登录页。
  4. 初始化完成后进入登录页，即可用默认管理员账号登录。
注意： 升级或重装时，不要手工删除安装目录下的数据文件，否则可能导致居民、调解、耕地等业务数据丢失。

2. 首次登录

项目 说明
默认账号 superadmin
默认密码 admin123
适用场景 仅适用于首次安装或尚未改密的环境；正式使用前建议立即修改默认密码。
推荐顺序：安装 → 首次登录 → 修改默认密码 → 进入系统信息确认版本 → 配置字典 → 配置界面 → 再开始正式录入或导入。

三、界面总览

1. 首页与左侧菜单

进入系统后，常见菜单包括：

首页 / 仪表盘

查看居民、耕地、待办等概览数据。

居民管理

居民查询、新增居民、人口增减上报、务工信息、风险排查。

特殊人群管理

低收入人员、政策享受记录、残疾人、两项补贴等专项台账。

机构管理

组织架构、党员管理、党费缴纳。

待办提醒

查看系统提醒、任务提醒、生日/事件提醒。

耕地管理

耕地查询、地块高清图。

人民调解

调解档案、申请书、记录、协议、案卷。

报表中心

按居民和字段组合生成上报名单。

系统设置

设置概览、数据库安全、通知配置、字典管理、界面设置、耕地图纸上传。

权限管理

角色管理、用户管理。

实用工具

维护常用文书模板，生成会议通知、证明等材料并导出 Word。

2. 右上角用户区

右上角头像区域一般提供以下入口：

  • 个人中心：查看和修改个人信息。
  • 系统信息：查看版本、运行模式、授权和备份情况。
  • 待办提醒：快速查看未读提醒和任务状态。
  • 退出登录：退出当前账号。

四、系统信息、授权与备份

进入路径：右上角头像 → 系统信息。

1. 运行模式：单机版与局域网版

模式 说明
单机版 数据仅限当前电脑访问，适合单人办公或单机使用。
局域网版 允许同一办公室网络内的其他电脑访问，适合多人协作；安装版通常需要有效授权。
  • “申请模式”是你想切换到的模式。
  • “实际模式”是系统当前真正生效的模式。
  • 如果提示“待重启生效”，说明配置已保存，但服务还没按新模式重新启动。
切换到局域网版后，系统会显示一个访问地址。其他电脑需要与主机连接在同一个办公室网络内，才能正常访问。

2. 授权文件导入

  1. 在系统信息里点击“导出授权申请文件”，把生成的申请文件交给授权方。
  2. 授权方根据申请文件生成与本机匹配的授权文件。
  3. 点击“导入授权文件”并选择授权文件。
  4. 导入成功后，重新确认申请模式 / 实际模式 / 是否需要重启。
局域网版仅适用于内部网络共享，不等于互联网远程访问。如需外网访问，应单独设计网络方案。

3. 数据备份

系统信息内可查看备份目录、最近备份时间、最近备份文件和备份数量；superadmin 可手动创建数据备份、修改备份设置和执行恢复。

  • 可开启或关闭自动备份，并设置自动备份间隔。
  • 可使用默认备份目录，也可由管理员指定备份目录。
  • 重要导入前建议先手动备份一次。
  • 导入居民、耕地、低收入、务工、风险排查等批量数据前，建议先确认备份已完成。
  • 如需从备份恢复，需要选择备份文件，并输入纸质记录上的恢复密钥（Recovery Key）验证。

4. 版本更新

当前系统按物理隔离要求使用，不通过软件自动联网检查或下载新版本。需要升级时，由管理员从指定渠道获取安装包，先完成数据备份，再在原安装目录上覆盖安装。

五、基础设置

1. 字典管理

进入路径：系统设置 → 字典管理。

字典管理决定了很多下拉项、业务分类和编号前缀，首次上线前建议优先配置。

优先级 建议先配置的字典
必须 村组、民族、政治面貌、文化程度、婚姻状况、兵役状况
按业务配置 调解档案前缀、纠纷类型、使用单位、低收入分类、残疾类别和等级
特别提醒： 对于“村组”“使用单位”“调解档案前缀”这类会参与识别或编号的字典，建议把名称和编码都维护完整。

2. 数据库安全

进入路径：系统设置 → 数据库安全。

  • 用于查看数据安全初始化状态、数据库加密状态和恢复密钥校验信息。
  • 首次启用或管理员交接时，应按页面提示打印并线下保管恢复密钥（Recovery Key）。
  • 换电脑、重装系统或恢复备份时，可能需要输入纸质记录上的恢复密钥。
恢复密钥是恢复数据的重要凭据，建议纸质保管，更不能保存在安装本软件的电脑上。

3. 界面设置

进入路径：系统设置 → 界面设置。

  • 可调整系统标题、Logo 显示方式、Logo 图片、Logo 文字、浏览器图标。
  • 建议首次安装后立即改成单位正式名称和图标。
  • 保存后若图标未立即变化，可按 Ctrl+F5 强制刷新。

4. 通知配置与耕地图纸

  • 通知配置：用于维护待办提醒相关的通知设置。
  • 耕地图纸上传：用于上传和维护“地块高清图”中查看的图片资料。

5. 权限管理

进入路径：权限管理 → 角色管理 / 用户管理。

  • 角色管理：用于配置某类人员可以访问哪些模块。
  • 用户管理：用于新建账号、分配角色、重置密码。
  • 建议先设计角色，再创建用户，避免每个用户都单独配置权限。

六、居民与特殊人群

1. 居民管理

居民管理是多数业务模块的基础，建议先把居民数据整理准确，再开展特殊人群、组织、调解等业务。

  • 居民查询：按姓名、身份证号、联系电话、村组、状态等条件筛选居民。
  • 新增居民：逐条录入居民基本信息。
  • 人口增减上报：记录人口变化情况。
  • 导入居民：通过 Excel 批量导入居民基础信息，导入前请先下载模板；导入入口仅超级管理员可见。

2. 居民导入建议

  1. 先确认“字典管理 → 村组”已配置完整。
  2. Excel 中身份证号、银行卡号等建议设置为文本格式。
  3. 尽量使用系统模板或按模板表头整理原始表。
  4. 先小批量导入验证，再导入全量。
如果导入后提示“村组未匹配”或“村组缺编码”，通常是字典管理尚未配置完整，请先检查“村组”字典。

3. 特殊人群管理

包括 低收入人员查询、残疾人查询、两项补贴 等。

  • 低收入人员支持新增、编辑、政策享受记录维护和批量导入。
  • 低收入批量导入按身份证号关联居民，必填字段包括身份证号、政策类型、开始日期。
  • 残疾人信息可基于居民基础数据维护，支持查看和维护两项补贴信息。
  • 建议先保证居民身份证号、姓名、家庭关系等基础字段准确。

4. 务工信息

进入路径：居民管理 → 务工信息。

  • 用于维护居民就业状态、务工记录、相关统计信息。
  • 适合做外出务工、返乡就业等台账管理。
  • 批量导入时按身份证号关联居民，不会自动新增居民基础信息；导入入口仅超级管理员可见。
  • 务工导入模板中，身份证号、变更类型、是否就业为必填项。

5. 风险排查

进入路径：居民管理 → 风险排查。

  • 用于维护预警对象、风险等级、预警内容和排查处理情况。
  • 可新增预警、查看详情、补充排查记录，并按需导出数据。
  • 批量导入采用统一导入界面：上传文件、字段映射、数据预览、导入结果。
  • 风险排查导入模板中，姓名、身份证号、预警时间为必填项；村组、户主姓名、风险等级、预警内容等为选填项。

七、机构管理

1. 组织架构

进入路径：机构管理 → 组织架构。

  • 用于维护现任班子、历届成员、职务信息、任期信息。
  • 建议统一职务名称、联系电话格式和任期填写口径。
  • 如果单位有固定称谓，建议先在内部确定命名规范再批量录入。

2. 党员管理

进入路径：机构管理 → 党员管理。

  • 支持党员查询、详情维护、转入转出记录、职务记录、家庭概况等信息。
  • 点击列表行可进入详情窗口，查看并维护党员资料。
  • 如果党员与居民基础数据关联，建议优先核对居民主档信息。

3. 党费缴纳

进入路径：机构管理 → 党费缴纳。

  • 用于维护党员党费缴纳记录，便于按人员、时间和缴费状态进行核对。
  • 建议先维护党员基础信息，再录入或核对党费记录。

八、人民调解

1. 办理流程

进入路径：人民调解。

  1. 先新建调解档案，系统会按前缀自动生成档案编号。
  2. 进入档案详情，填写申请书信息和当事人信息。
  3. 根据实际调解过程新增调解记录。
  4. 达成协议后再填写协议书，系统会把档案状态更新为已完成。
  5. 如需归档打印或留底，可在打印预览中选择另存为 PDF 档案。

2. 调解案卷

  • 案卷功能用于查看调解相关材料，归档时建议通过打印预览另存为 PDF。
  • 如果需要纸质留存，可直接从打印预览页面打印。
  • 删除档案前，建议先确认附件和案卷是否已备份留存。

九、耕地管理

1. 耕地查询

进入路径：耕地管理 → 耕地查询。

  • 可按承包户、地块编号、村组、地块状态等条件筛选。
  • 查询结果可在“地块明细”和“按户汇总”之间切换。
  • 进入详情后可查看地块面积、四至、流转情况、确权情况、变更记录。
  • superadmin 可执行导入台账、承包户变更、地块调换、综合变更等管理操作。
  • 导入台账前建议先下载模板，并确认村组、承包户姓名、身份证号、地块编号和面积等信息整理准确。

2. 地块高清图

进入路径：耕地管理 → 地块高清图。

  • 用于查看已上传的地块高清图。
  • 图片资源通常由管理员在“系统设置 → 耕地图纸上传”中维护。
  • 建议上传清晰、命名规范的 JPG/PNG 图片，便于后续核对。

十、报表与提醒

1. 报表中心

进入路径：报表中心。

  • 先选居民，再选字段，按需拼装导出内容。
  • 适合快速生成名单、上报名单、专项台账预览。
  • 风险排查字段中可选择“排查次数”，用于统计某个预警对象已登记的排查记录数量。
  • 建议先想清楚“报给谁看”和“要哪些列”，避免重复导出。

2. 待办提醒

进入路径：待办提醒，或通过右上角快捷入口进入。

  • 支持查看全部、未读、任务、生日提醒、事件提醒等视图。
  • 支持批量已读、批量未读、批量删除、全部已读。
  • 有权限的用户可新建提醒并维护提醒详情。

3. 实用工具

进入路径：实用工具。

  • 可维护常用文书模板，例如会议通知、证明材料等。
  • 支持保存模板、应用模板、覆盖模板、删除自建模板，并导出 Word 文件。
  • 内置模板可直接套用，不建议直接覆盖，可另存为单位自己的模板。

十一、数据安全与升级

1. 备份建议

  • 大批量导入前先手动备份。
  • 升级安装包前先备份。
  • 重要集中修改（字典、权限、耕地、调解台账）前先备份。

2. 升级建议

  • 升级前记录当前版本和最近备份时间。
  • 安装新版前不要手工删除数据目录或数据文件。
  • 升级完成后，优先检查：登录、首页、系统信息、居民查询、核心业务页面。
建议的升级后核对顺序： 登录 → 系统信息 → 居民查询 → 待办提醒 → 人民调解 → 耕地管理 → 报表中心。

十二、常见问题

1. 为什么切到局域网版后，其他电脑还是打不开？

先检查三件事：是否已经导入有效的局域网授权、系统信息中“实际模式”是否已经变成局域网版、是否已经按提示完成重启。

2. 为什么系统提示有备份，但我不敢升级？

这是正常的谨慎做法。建议在系统信息里再手动创建一次备份，确认最新备份时间后再升级。

3. 为什么导入居民或耕地时老报字典问题？

因为导入依赖基础字典。只要村组、状态、分类等字典没配完整，导入就可能失败或出现未匹配提示。

4. 默认账号是不是永远都是 superadmin / admin123？

不是。它只适用于首次安装或尚未改密的环境。只要单位已经改过密码，就应以当前实际密码为准。

5. 为什么不建议手工删除数据文件？

因为居民、特殊人群、机构、调解、耕地、待办等业务数据都保存在本机数据文件中。手工删除数据文件，相当于直接删掉业务数据。

6. 报表中心里为什么查不到某些居民？

先检查居民基础数据是否存在，再检查筛选条件是否过严，例如“是否残疾人”“是否就业”“特殊人群”等是否已限制结果。

7. 为什么有些页面看不到批量导入按钮？

居民、务工、低收入、风险排查、耕地等批量导入入口已收紧到超级管理员。普通业务账号看不到入口属于正常权限控制。

8. 下载导入模板时还能自己选保存位置吗？

可以。模板下载会调用系统保存对话框，选择保存位置后再生成文件；如果取消保存，不会生成文件。

9. 系统会不会自动联网检查新版本？

不会。当前正式使用按物理隔离要求执行，系统不提供在线检查和在线下载。升级包由管理员从指定渠道取得，备份后覆盖安装。

本指南适用于当前版本智慧村务系统安装版。如后续菜单、字段或业务流程调整，请同步更新本页内容。
